# Wiki Query

위키를 검색하여 질문에 답변하고, 새로운 통찰이 있으면 위키에 역기록하는 질의응답 스킬.

## 워크플로우

### 1. 인덱스 스캔

`wiki/index.md`를 읽어 카테고리별 페이지 목록을 파악한다. 질문의 키워드와 관련된 카테고리와 페이지를 식별한다.

### 2. 관련 페이지 검색

두 가지 전략을 병행한다:

- **인덱스 기반**: index.md에서 질문과 관련된 페이지를 선택한다
- **전문 검색**: Grep 도구로 `wiki/` 디렉토리에서 질문의 핵심 키워드를 검색한다

### 3. 페이지 읽기 및 정보 수집

관련 페이지를 읽고 질문에 답할 수 있는 정보를 수집한다. 각 정보의 출처 페이지를 기록한다.

### 4. 답변 생성

수집한 정보를 종합하여 답변한다. 답변의 끝에는 반드시 아래 두 블록을 포함한다. 이 형식은 Obsidian에서 백링크 추적과 쿼리 감사(어떤 페이지가 실제로 참조되었는지)를 가능하게 하므로 생략하지 않는다.

```markdown
{답변 본문 — 본문 안에서는 [[링크]]를 인라인으로 쓰지 않는다}

**참조한 위키 페이지:**
- [[페이지1]] — {해당 페이지에서 가져온 핵심 정보 한 줄}
- [[페이지2]] — {해당 페이지에서 가져온 핵심 정보 한 줄}

**역기록:** {새 통찰이 있으면 생성한 페이지명, 없으면 "없음 — 기존 정보의 종합이므로 역기록하지 않음"}
```

### 5. 역기록 (write-back)

여러 페이지의 정보를 교차 분석하여 **기존 위키 어디에도 없는 새로운 통찰**이 발생한 경우에만 역기록한다:

1. `wiki/syntheses/`에 새 페이지를 생성한다
2. frontmatter에 `generated-from: query`를 추가한다
3. `wiki/index.md`에 등록한다

기존 정보를 재구성한 답변은 역기록하지 않는다. 역기록 여부와 그 판단 근거는 답변 끝에 항상 명시한다.

### 6. 로그 기록

답변을 사용자에게 보여준 뒤, `.wiki-log.md`에 쿼리 기록을 남긴다. 이 기록이 있어야 나중에 Lint 시 "어떤 질문이 들어왔고 어떤 페이지가 참조되었는지" 추적할 수 있다. 답변 생성과 로그 기록은 하나의 연산이다 — 답변만 하고 로그를 빠뜨리면 연산이 미완료된 것이다.

```markdown
## [YYYY-MM-DD] query | {질문 한 줄 요약}
- 참조: {읽은 페이지 목록}
- 역기록: {있으면 새 페이지명, 없으면 "없음"}
```

## 답변 원칙

- 위키에 있는 정보만으로 답한다. 외부 지식을 섞지 않는다.
- 위키에 정보가 없으면 "위키에 해당 정보가 없습니다"라고 답하고, 소스 수집을 제안한다.
- 모순된 정보가 있으면 두 출처를 모두 제시한다.
- 답변은 간결하고 구조적으로 작성한다.
